# Mitsurugi Kamui Hikae - tools

Ковырял русик для Mitsurugi Kamui Hikae (движок ZAKURI / ZenithBlue) и по ходу разобрал почти все её форматы. Тут лежат питоновые тулзы для всего: архив, текст, шрифт, текстуры. Может кому пригодится - игра старая, инструментов под неё толком нет.

Нужен Python 3. Для текстур ещё `pillow` (`pip install pillow`).

## Что вообще как устроено

Всё лежит в одном `data0.kpk` (~100 МБ): просто склеенные файлы, каждый либо пожат zlib, либо как есть, выровнено по 8 байт. Списка имён внутри нет - **таблица файлов (смещения/размеры) зашита в `mitsurugi.exe`** (`.rdata` @ `0x4BFE50`, записи по 24 байта, ключ = хеш имени). Игра бинпоиском по ключу находит запись и фисикает на абсолютное смещение в kpk.

Отсюда главное правило: **поменял размер файла - пересобирай kpk и exe вместе**. Все тулзы это учитывают.

Где какой текст:
- **Диалоги/субтитры** - UTF-16LE строки в файлах типа `04` (`.b04`). Правится как текст.
- **Меню/туториалы/ачивки/описания** - UTF-16LE строки в `.e3` (сцены). Тоже текст.
- **Кнопки главного меню и часть HUD** - предрендеренные картинки-слова в `.tex`/DDS-атласах. Только перерисовкой.
- **Шрифт** - `.fm` (метрики глифов) + атласы `.tex`. Кириллицы в оригинале нет, надо дорисовывать.

## kpktool.py - архив

```
python kpktool.py unpack data0.kpk mitsurugi.exe out/     # распаковать всё + manifest.json
python kpktool.py verify data0.kpk mitsurugi.exe          # проверка round-trip (должно быть IDENTICAL=True)
python kpktool.py replace data0.kpk mitsurugi.exe <ключ_hex> новый.bin out.kpk out.exe
```
`replace` сам пересобирает kpk и патчит таблицу в exe. На выходе пара - ставить в игру только вместе.

Из кода: `unpack_exact()`, `repack_multi(unpacked, {ключ: байты})`, `patch_exe_directory(exe, patches)`.

## b04tool.py - диалоги

```python
import b04tool
m = b04tool.parse(open('файл.b04','rb').read())
print(m['strings'])                 # список реплик
b04tool.set_strings(m, новые)       # столько же строк, сколько было
open('out.b04','wb').write(b04tool.build(m))
```
Строки лежат в блоке, на который указывает мастер-таблица (`[12]`..`[13]`). При смене длины тулза двигает хвостовые смещения сама.

## e3tool.py - меню

То же самое для `.e3` (меню, туториалы, ачивки, описания апгрейдов):
```python
import e3tool
m = e3tool.parse(open('b0001.e3','rb').read())
e3tool.set_strings(m, новые)
open('out.e3','wb').write(e3tool.build(m))
```
Блок строк - `[hdr@0x68 .. hdr@0x6c)`. При ресайзе двигаются хвостовые офсеты (`SHIFT_OFFS`). Часть `.e3` без текста - `parse` вернёт `has_block=False`, их не трогаем.

## fmtool.py - шрифт

`.fm` - метрики: 3 страницы (Main_46, Main_28, System_28), у каждой свой атлас. Таблица глифов - записи по 8 байт (`cp, U, V` u16, `w, h` u8).
```python
import fmtool
m = fmtool.parse(open('all.fm','rb').read())
recs0 = fmtool.page_records(m, 0)   # глифы страницы 0
# добавил/поменял записи -> собрать три страницы заново:
fmtool.set_pages(m, recs0, recs1, recs2)
open('out.fm','wb').write(fmtool.build(m))
```
Подводные камни (важно, иначе игра падает/мажет):
- В таблице глифов в конце есть **одна лишняя пустая запись** (паддинг) - `build` её добавляет.
- После таблицы строк идёт **48-байтовый хвост нулей** - тоже обязателен, `build` ставит.
- Координаты глифов `U/V/w/h` указывают в атлас соответствующей страницы.

## textool.py - текстуры и атласы

```
python textool.py tex2png in.tex out.png        # посмотреть атлас
python textool.py png2tex orig.tex in.png out.tex   # вернуть, сохранив исходный заголовок
python textool.py dds2png in.dds out.png         # DDS-атласы (DXT1/5)
```
`.tex` - заголовок (44 или 18 байт) + пиксели BGRA, размеры в `@12/@14`. Нюансы по конкретным атласам (некоторые хранятся с альфа-каналом, некоторые цвет-кеят мадженту, часть перевёрнута по вертикали) - зависят от слота; смотри глазами через `tex2png`.

## inject.py - собрать всё разом

Склейка: распаковать → подменить ВЕСЬ текст по словарю «английский→русский» (сам пробегает все `.b04` и `.e3`) → плюс подменить бинарники (шрифт, атласы) → собрать kpk и пропатчить exe → проверить.

```
python inject.py data0.kpk mitsurugi.exe en2ru.json e3_dir/ out.kpk out.exe \
    AB4CD424=all_ru.fm 63de01d2=Main_Font_28_ru.tex
```
- `en2ru.json` - `{"English string": "Русская строка", ...}`. Применяется и к диалогам, и к меню.
- `e3_dir/` - папка с распакованными `.e3` (из `kpktool unpack`).
- дальше пары `ключ=файл` - бинарная замена (шрифт `.fm`, атласы `.tex`). Ключи см. в `FORMATS.md`.
- на выходе печатает `verify_mismatch` - должно быть 0.

Из кода: `inject.build_patch(kpk, exe, out_kpk, out_exe, en2ru=..., e3_dir=..., binary_replacements={ключ: байты})`.

## fontgen.py - дорисовать кириллицу в атлас

Рендерит глифы из TTF прямо в атлас (в нужном формате) и дописывает их в `.fm`. Та самая «кнопка» вместо ручной возни.
```
python fontgen.py all.fm atlas.tex font.ttf <логич.страница 0|1|2> out.fm out.tex [magenta|alpha] [flip] [cyr | cp1,cp2,...]
```
- логич. страница: 0 = Main_46, 1 = Main_28 (субтитры), 2 = System_28 (меню).
- формат: Main_28 - `magenta`, System_28 - `alpha flip` (см. FORMATS.md).
- `cyr` = весь русский алфавит (А-я + Ёё), либо список кодов через запятую.
- кладёт глифы в свободные ряды атласа; если не лезет - сначала увеличь атлас. Лимит всего глифов ~275.

## gui.py - графическая студия

```
python gui.py
```
Открываешь kpk+exe - получаешь список всех файлов (фильтр, тип, размер). По клику:
- `.b04`/`.e3` - правишь строки прямо в окне (выбрал строку, поменял, Apply line);
- `.tex`/`.dds` - смотришь атлас картинкой, можешь сохранить в PNG;
- остальное - hex/инфо.
Кнопки: **Build patch** (собирает kpk+exe с твоими правками), **Diff with...** (сравнить с другим kpk), **Export PNG**.

## analyze.py - разбор игры

```
python analyze.py data0.kpk mitsurugi.exe [dump_text.json]
```
Сводка: сколько файлов и каких типов, сколько текста (строк), какие шрифты (страницы/глифы), инвентарь атласов. С третьим аргументом - выгружает ВЕСЬ текст игры в JSON (удобно как основа для перевода).

## kpkdiff.py - сравнить две сборки

```
python kpkdiff.py A.kpk A.exe B.kpk B.exe
```
Показывает, какие файлы изменились/добавились/удалились, и для текстовых - сколько строк поменялось. Удобно проверить свой русик против оригинала или сравнить версии игры.

## Типичный цикл

1. `kpktool.py unpack` → нашёл нужный файл по ключу в `manifest.json`.
2. правишь: `b04tool`/`e3tool` для текста, `fmtool`+`textool` для шрифта/атласов.
3. собираешь всё через `inject.py` (или вручную `kpktool.repack_multi` + `patch_exe_directory`, или `replace` для одного файла).
4. кладёшь kpk+exe в игру **парой**, бэкап оригиналов держишь.

## FORMATS.md

Отдельным файлом - подробный разбор форматов (структура `.fm`, форматы атласов, ключи файлов, капканы вроде хвоста `.fm` и центровки субтитров). Читай ПЕРЕД тем как лезть в шрифт - там описаны грабли, на которых игра падает.
